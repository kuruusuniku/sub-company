CEOを召喚するコマンド。夜間の自律実行や、大型タスクの戦略的な計画・実行を行う。

## 引数による分岐

引数 `$ARGUMENTS` を確認し、以下のいずれかに分岐する：

### 「夜間モード」「夜に」「寝てる間に」「overnight」の場合 → 夜間タスク設定

### 「ステータス」「結果」「朝の報告」の場合 → 夜間タスクの結果確認

### それ以外（具体的なタスク内容）の場合 → CEO 即時実行

---

## 夜間タスク設定モード

ユーザーに夜間タスクの内容を聞き、スケジュールタスクとして登録する。

### ステップ1：タスク内容の確認

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🌙  CEO 夜間モードの設定
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
CEO が夜間に自律的にタスクを実行します。

朝起きたら結果レポートが届いています！
CEOが寝ずに働いてくれるので、
あなたはぐっすり眠ってください。

何を実行しますか？
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

ユーザーに以下を聞く（チャットで直接質問する）：
- タスク内容（自由記述）
- 実行時刻（デフォルト：深夜2:00）
- 繰り返し（一度だけ / 毎日 / 毎週）

### ステップ2：タスクの詳細設計

CEO として依頼内容を分析し、実行計画を立てる：

1. 依頼を複数のサブタスクに分解する
2. 各サブタスクの担当部署を決定する
3. 実行順序と依存関係を整理する
4. 成果物の品質基準を設定する

計画を `company/ceo/plans/night-plan-YYYY-MM-DD.md` に保存する。

### ステップ3：夜間タスクの保存と実行方法

夜間タスクの実行計画を `company/ceo/plans/night-plan-YYYY-MM-DD.md` に保存する。

次に、以下の方法で夜間実行を設定する：

**方法A：Claude Code の `--resume` を使う方法（推奨）**

ユーザーに以下のコマンドを案内する：
```bash
# 指定時刻に実行（例：深夜2時）
echo 'claude --resume --print "company/ceo/plans/night-plan-YYYY-MM-DD.md を読んで実行して"' | at 02:00
```

**方法B：cron を使う方法**

```bash
# crontab に登録（例：毎日深夜2時）
(crontab -l 2>/dev/null; echo "0 2 * * * cd $(pwd) && claude --print 'company/ceo/plans/night-plan-YYYY-MM-DD.md を読んで実行して' >> company/ceo/reports/cron-log.txt 2>&1") | crontab -
```

**方法C：即時実行（今すぐ CEO に任せる）**

ユーザーが「今すぐやって」と言った場合は、CEO サブエージェントを spawn して即時実行する。

夜間タスクの prompt テンプレート（計画ファイルに含める）：

```
あなたは [会社名] の CEO です。夜間自律実行モードで動作しています。

## 実行計画
[ステップ2で作成した計画の内容]

## 実行手順

1. `company/secretary/config.md` を読み込む
2. 計画に従い、各部署にサブエージェント（Agent ツール）で命令を出す
3. 各部署の成果物を確認・レビューする
4. 品質に問題があれば修正指示を出して再実行する（最大2回）
5. 全タスク完了後、CEO レポートを作成する：
   `company/ceo/reports/night-report-YYYY-MM-DD.md`
6. GitHub 連携済みの場合は Issue を作成・完了処理する

## CEO レポートの構成
- 実行したタスクの一覧と結果
- 各部署の成果物ファイルパス
- 品質レビューの結果
- 特記事項・懸念点
- 次に推奨するアクション

## 注意事項
- ユーザーは寝ています。確認が必要な場合は Issue に「要確認」ラベルを付けて記録し、朝に確認してもらう
- 判断に迷う場合は安全な選択をし、理由を記録する
- 長時間実行になる場合はこまめにログを残す
```

```bash

```

## CEO 即時実行モード

ユーザーが具体的なタスクを指定した場合、CEO を即座に spawn して実行する。
処理は `/ask` のステップ2C と同じ CEO サブエージェントの仕組みを使う。

ただし `/ceo` 経由の場合は以下が追加される：
- CEO が実行前に計画を `company/ceo/plans/plan-YYYY-MM-DD-NNN.md` に保存する
- 計画をユーザーに表示し、承認を得てから実行する
- 完了後に CEO レビューレポートを `company/ceo/reviews/` に保存する

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🏢  CEO を召喚しました
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[依頼内容] について CEO が分析中です。

実行計画ができ次第ご報告いたします。
少々お待ちください。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```
