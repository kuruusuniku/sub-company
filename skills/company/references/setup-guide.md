# 子会社設立ガイド

## 設立手順

### ステップ1：導入演出

以下の導入メッセージを表示してから、ヒアリングを開始する：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
　　🏢  専属子会社 設立システム  🏢
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

あなたを全力でサポートする専属の子会社を
これより設立いたします。

全部署が連携し、あなたの目標達成に向けて
24時間体制で稼働します。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### ステップ2：ヒアリング（2点のみ）

チャットで以下の2点を確認する。**部署の選択は不要**——全部署を設立する：

```
設立にあたり、2つだけ教えてください！

① あなたのメイン活動・目標は何ですか？
  （例：SaaS開発、EC事業、フリーランス、
   コンテンツ制作、スタートアップ など）

② 子会社代表秘書のお名前をつけてください
  （例：さくら、凛、Alex）
  （省略した場合は「秘書」になります）
```

### ステップ3：進捗演出つきでファイル生成

以下を順番に生成し、生成のたびに進捗を表示する：

```
[設立中...] 秘書室の準備中... ✅
[設立中...] PM部を配置中...  ✅
[設立中...] 開発部を配置中... ✅
[設立中...] マーケ部を配置中... ✅
[設立中...] 営業部を配置中... ✅
[設立中...] 経営企画部を配置中... ✅
```

#### 生成ファイル一覧

**`company/secretary/config.md`**（最初に生成）
```markdown
# 秘書設定ファイル

このファイルは秘書の設定を保存します。直接編集することで振る舞いを変更できます。

## 基本設定

- **name**: [入力された名前 / 未入力なら「秘書」]
- **tone**: ていねい  # ていねい / カジュアル / フレンドリー
- **mission**: [ユーザーのメイン活動内容]
- **created_at**: YYYY-MM-DD

## 挨拶設定

- **greeting**: [name]でございます。
- **sign_off**: 以上、[name]よりご報告いたしました。
```

**`company/dashboard.md`**
```markdown
# 🏢 専属子会社ダッシュボード

> 代表秘書：[秘書名] ｜ 設立日：YYYY-MM-DD

## 🎯 ミッション
[ユーザーのメイン活動]のご支援

## 配置部署
| 部署 | 役割 |
|------|------|
| PM部 | プロジェクト・スケジュール管理 |
| 開発部 | 技術調査・システム設計 |
| マーケ部 | 戦略・コンテンツ・告知 |
| 営業部 | 提案書・台本・クライアント対応 |
| 経営企画部 | 戦略・分析・売上管理 |

## 使い方
- `/ask [ご依頼内容]` — 秘書が各部署へ命令を出して対応
- `/report weekly` — 週次レポート生成
- `/company` — このダッシュボードを表示
- `/company rename [名前]` — 秘書の名前を変更
```

---

## リセット手順

ユーザーが「リセット」を選択した場合：
1. `company/` フォルダを `company_backup_YYYYMMDD/` としてバックアップ
2. `company/` フォルダを削除
3. 子会社設立フローを最初から開始
